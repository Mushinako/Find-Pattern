How to use this?

Input the number of test cases as an integer, optionally in "E-notation". E.g., a million can be represented as "1000000" or "1e6".

Please add 0's before decimal points when necessary, i.e., ".13e6" is not valid, but "0.13e6" is.

Non-integers will be rounded up.

Be aware that this algorithm is \(O(b^p)\), thusly the amount of time required expoentially when ball/period increases.

To avoid integer overflow, it is recommeded that the parameters do not exceed 100[Citation Needed]. To finish the calculations before the heat death of the universe, limit the parameters to below 10.

This script works on Chrome v67+, Firefox v60+, and the beloved Internet Explorer 1+. Testing on Safari 11+ is incomplete.
